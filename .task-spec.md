## Rule coverage (the contract)
Deny (hard block + log reason): `rm -rf` (esp. `/`, `~`, `.`), `sudo rm`,
`chmod 777`, `chown -R root`, `git reset --hard`, `git clean -fd`,
`git push --force` (but ALLOW `--force-with-lease`), `git stash clear`,
`git filter-branch`, remote branch delete (`git push origin --delete`,
`git push origin :branch`), `> /dev/sd*`, `mkfs`, `dd of=/dev/`, fork bombs.
Ask (require confirmation, not hard-deny): borderline ones — bare `git reset`,
`git checkout -- .`, `npm publish`, `rm -r` without `-f`. Keep it data-driven so
rules are editable without code changes.

## Extension behavior
- Hook the bash tool invocation; match the command string against the rules.
- On a `deny` match: block execution, return/log the matched rule's reason.
- On an `ask` match: surface a confirmation rather than a hard block.
- No match: allow.
Keep `--force-with-lease` explicitly allowed even though `--force` is denied.
